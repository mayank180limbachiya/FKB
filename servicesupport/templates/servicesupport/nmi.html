PC50 Alarm / 971 NMI occured / SLC Error


About Alarm:

When I/O Link communication fails, System Alarm is issued with one of the following messages. The Message changes according to the series and edition of PMC management Software, However, Its Meaning and the process of troubleshooting are common.


Input:

Here "a1b1" for Channel-1, "a2b2" for Channel-2 for Earlier values (before Alarm Register value)
Here "x1y1" for Channel-1, "x2y2" for Channel-2 for Later values (after Alarm Register value)
Enter "a" "x" Value: Enter "b" "y" Value:

Output:

Xn#7 Values
Bit Xn#7 Bit 6 Bit 5 Bit 4 Bit 3 Bit 2 Bit 1 Bit 0
0 0 0 0 0 0 0 0
5-7 Meaning less Parity error in I/O RAM, Exchange PMC controller module or the board with i/o link connector SLC,I/O link controller LSI, detect Parity error in internal RAM . Exchange PMC control module , or the board with i/o link connector Slave Did not respond. Link was discconected slave device detect an error, see Yn for more details bad data was received : xn#1 is 1,xn#0 will always be so check xn#1 first

Yn#7 Values
Bit Yn#7 Bit 6 Bit 5 Bit 4 Bit 3 Bit 2 Bit 1 Bit 0
0 0 0 0 0 0 0 0
Slave Detected watchdog error or parity error.(not effective if xn#1 is 0) slave raised system alarm othger that one about I/O link. Check what system alarm was raised at slave.(not effective if xn#1 is 0) Slave Detected Communication error.(not effective if xn#1 is 0) number to identify slave: The Value is "group number+1" where the 1st slave is group 0, the value 4 means the group 3, which is 4th slave form master

When slave detected communication error(in case xn#1 is 1), this value shows the slave number and the details of the error. otherwise(in case xn#1 is 0), this byte will be the slave number of which master communicated last, or of which master was trying to commuincated last, or of which master was trying to communicate.But this slave number does not mean the very slave with problem. For example , in case that connection between 1st slave and 2nd slave was cut when master was trying to communicate with 3rd slave, this byte will show the number for 3rd slave.

  • *PC051 IOLINK ER2 CHn:v1:n1:x1:y1
  • *PC050 IOLINK CH1 a1b1-x1y1:x2y2

  • *PC050 NMI SLC(CHn) x1:y1 x2:y2

  • *PC150 NMI SLC x1:y1
  • *971 NMI OCCURRED IN SLC (Note)
  • *SLC ERROR x1(y1) : PC050
  • *SLC ERROR x1 : PC050

(Note : 971 NMI OCCURRED IN SLC has the register value "x1y1" in memory dump. The SLC's register value is not shown in the error message itself. however register value shown in dumo display.)


CHn (n>0): Channel of communication failure. The messages without "CHn" menas the error war detected in channel 1.
GRPn (15>n>0) : Group of I/O link slave device, which has a possibility that some problem occurs.
n1 : Means the count of communication error. Retry Counter.
"a1b1","x1y1","a2b2","x2y2": The register values of SLC, which is I/O link communication controller LSI.

Earlier Later
CH1 a1b1 x1y1
CH2 a2b2 x2y2

The register values are clues certainly, however, they may not be enough evidence to find out the cause of trobles because of folloeing reasons:
  • These values are not the values at the instant when the first communication error was detected,but are the values when the communication disconneted finally. this means that the values may not reflect the direct cause, and can reflect its side effect.
  • In case of hardware malfuction or electric noises, the register value may become improper value
  • The same causes may show different register value dpending on the timing of error.
Therefore, it is important to analyze not only register value, but also the situation how the alarm occured.
